## 6.2 Docker Compose 구성 (Production)

### 서비스 구성

| 서비스 | 이미지 | 포트 | 설명 |
|--------|--------|------|------|
| hopenvision-api | hopenvision-api | 9050:8080 | Spring Boot 백엔드 |
| hopenvision-web | hopenvision-web | 4060:80 | 사용자 포털 (Nginx + React) |
| hopenvision-admin | hopenvision-admin | 4061:80 | 관리자 콘솔 (Nginx + React) |

### 네트워크

```
database-network (external)
 └── PostgreSQL (공유) + hopenvision-api
```

- `database-network`: 외부 네트워크 — 다른 서비스와 PostgreSQL 공유
- 프론트엔드 → API: Nginx 리버스 프록시 (`/api/**` → `hopenvision-api:8080`)

### Nginx 설정 (프론트엔드)

| 설정 | 값 |
|------|-----|
| SPA 라우팅 | 모든 요청 → `/index.html` (try_files) |
| API 프록시 | `/api/` → `http://hopenvision-api:8080` |
| 정적 파일 캐싱 | JS/CSS/이미지: 1년 |
| Gzip 압축 | 활성화 |
| 보안 헤더 | X-Frame-Options, X-Content-Type-Options |
| 업로드 제한 | 최대 20MB |

### 헬스체크

```
hopenvision-api:
  healthcheck:
    test: curl -f http://localhost:8080/api/health
    interval: 30s
    timeout: 10s
    retries: 3
```

## 6.3 Dockerfile

### Backend (api/Dockerfile)

```
Multi-stage Build:
1. gradle:jdk17        → Gradle 빌드 (bootJar)
2. eclipse-temurin:17  → JRE 런타임 (non-root user)

Port: 8080
```

> Alpine 이미지는 ARM64 (Apple Silicon) 미지원 — 비-Alpine 베이스 사용

### Frontend (web-user/Dockerfile, web-admin/Dockerfile)

```
Multi-stage Build:
1. node:20-alpine  → npm install + Vite 빌드
2. nginx:1.27      → 정적 파일 서빙

Port: 80
```

## 6.4 CI/CD (GitHub Actions)

### 트리거

```
prod 브랜치 push → 자동 배포
```

### 파이프라인

```
1. Checkout 코드
2. Backend 빌드 (Gradle)
3. Frontend 빌드 (npm)
4. Docker 이미지 빌드
5. 서비스 배포 (docker compose up -d)
```

### 러너

- Self-hosted Runner (macOS)
- 직접 Docker Compose 실행

## 6.6 브랜치 전략

```
main (개발)
  │
  ├── feature/* → main (PR merge)
  │
  └── main → prod (merge → 자동 배포)
```

- `main`: 개발 브랜치 (PR 기반 merge)
- `prod`: 운영 브랜치 (push 시 GitHub Actions 자동 배포)

## 6.7 배포 체크리스트

### 배포 전

- [ ] `main` 브랜치에서 빌드 성공 확인
- [ ] 테스트 통과 확인 (`./gradlew test`)
- [ ] DB 마이그레이션 스크립트 준비 (운영은 DDL `none`)
- [ ] 환경 변수 확인 (`.env` 파일)
- [ ] 프론트엔드 빌드 성공 (`npm run build`)

### 배포 후

- [ ] 헬스체크 확인 (`/api/health`)
- [ ] API 응답 확인 (`/api/exams`)
- [ ] 프론트엔드 페이지 로딩 확인
- [ ] 로그 모니터링 (`docker logs hopenvision-api`)
- [ ] DB 마이그레이션 적용 확인

## 6.9 트러블슈팅

### 자주 발생하는 문제

| 문제 | 원인 | 해결 |
|------|------|------|
| 포트 충돌 | 다른 서비스가 같은 포트 사용 | `docker ps`로 확인 후 포트 변경 |
| Alpine ARM64 실패 | gradle/JRE Alpine 이미지 ARM64 미지원 | 비-Alpine 이미지 사용 |
| gradlew 실행 오류 | Windows CRLF + 잘못된 JVM 옵션 | Dockerfile에서 `sed`로 수정 |
| DB 연결 실패 | database-network 미생성 | `docker network create database-network` |
| VITE_API_URL 오류 | 빈 문자열 처리 | `??` (nullish coalescing) 사용 |
