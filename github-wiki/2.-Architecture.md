## 2.2 백엔드 아키텍처

### DDD 패키지 구조

```
com.hopenvision/
├── config/                  # 공통 설정
│   ├── CorsConfig           # CORS 설정
│   ├── SecurityConfig       # Spring Security + API Key 필터
│   └── AdminController      # 관리자 인증 검증
│
├── exam/                    # 시험/채점 도메인 (Core)
│   ├── controller/
│   │   ├── ExamController           # 시험 CRUD + 채점
│   │   ├── ExamApplicantController  # 응시자 관리
│   │   ├── ExamImportController     # Excel/JSON 가져오기
│   │   ├── StatisticsController     # 통계 분석
│   │   ├── QuestionBankController   # 문제은행
│   │   ├── QuestionSetController    # 문제세트
│   │   └── ExamNoticeController     # 공지사항
│   ├── dto/                 # Request/Response DTO
│   ├── entity/              # JPA Entity
│   ├── repository/          # Spring Data JPA Repository
│   └── service/             # 비즈니스 로직
│
└── user/                    # 사용자 도메인 (Supporting)
    ├── controller/
    │   └── UserController   # 사용자 채점 API
    ├── dto/
    ├── entity/
    ├── repository/
    └── service/
```

### 계층 구조

```
Controller (REST API)
    ↓ DTO
Service (Business Logic)
    ↓ Entity ↔ DTO (MapStruct)
Repository (Spring Data JPA)
    ↓
PostgreSQL
```

## 2.3 프론트엔드 아키텍처

### 모노레포 구조 (npm workspaces)

```
hopenvision/
├── package.json              # 루트 (workspaces 선언)
├── web-shared/               # @hopenvision/shared
│   └── src/
│       ├── api/client.ts     # Axios 인스턴스 + createApiClient 팩토리
│       ├── types/common.ts   # ApiResponse<T>, PageResponse<T>
│       ├── types/exam-constants.ts  # 시험 유형, 과목 유형, 문항 유형
│       └── index.ts          # barrel export
├── web-user/                 # 사용자 앱 (채점)
│   └── src/
│       ├── api/userApi.ts
│       ├── components/       # UserLayout, OMRCard, QuickInputCard
│       ├── pages/            # UserExamList, UserAnswerForm, UserScoreResult, UserHistory
│       └── types/user.ts
└── web-admin/                # 관리자 앱 (시험관리)
    └── src/
        ├── api/              # adminClient.ts, examApi.ts, applicantApi.ts, statisticsApi.ts
        ├── auth/             # AuthContext, AuthGuard, useAuth
        ├── components/       # AdminLayout, ApplicantModal
        ├── pages/            # ExamList, ExamForm, AnswerKeyForm, ExcelImport, Statistics
        └── types/            # exam.ts, applicant.ts, statistics.ts
```

### 공유 코드 (`@hopenvision/shared`)

- **API Client**: Axios 인스턴스 팩토리 — 환경별 baseURL 자동 설정
- **공통 타입**: `ApiResponse<T>`, `PageResponse<T>` 래퍼
- **상수**: 시험 유형, 과목 유형, 문항 유형 매핑

## 2.5 보안 아키텍처

### 인증 방식

| 대상 | 방식 | 설명 |
|------|------|------|
| 관리자 | API Key | `X-Api-Key` 헤더로 전달, SecurityConfig에서 필터링 |
| 사용자 | Google OAuth + JWT | Google 로그인 → JWT 토큰 발급 |

### 보안 적용 사항

| 항목 | 구현 |
|------|------|
| CORS | 허용 도메인 명시적 설정 (와일드카드 금지) |
| API 인증 | 관리자 API Key + 사용자 JWT |
| 파일 업로드 | 파일명 검증 (Path Traversal 방지) |
| 보안 헤더 | X-Frame-Options, X-Content-Type-Options (Nginx) |
| 업로드 제한 | 최대 20MB |
| 민감 정보 | 환경변수로 관리, 소스코드 하드코딩 금지 |

## 2.6 네트워크 구성

```
┌──────────────────────────────────────────────┐
│              database-network (external)       │
│                                               │
│  ┌───────────┐  ┌───────────┐  ┌───────────┐│
│  │ postgresql │  │ hopenvision│  │  (다른     ││
│  │  :5432    │  │   -api     │  │  서비스)   ││
│  └───────────┘  └───────────┘  └───────────┘│
└──────────────────────────────────────────────┘
```

- PostgreSQL은 `database-network` 외부 네트워크에서 다른 서비스와 공유
- 프론트엔드 컨테이너는 Nginx 리버스 프록시를 통해 API 접근
- 외부 직접 DB 접근 차단

## 2.8 확장성 고려사항

### 현재: 모듈러 모놀리스

- DDD 기반 도메인 분리 (exam, user)
- npm workspaces 기반 프론트엔드 분리 (shared, user, admin)
- 도메인 간 결합도 최소화

### 향후 가능한 확장

- 배치 시스템 분리 (통계 집계, 순위 계산)
- 문제은행 서비스 독립
- 실시간 알림 (WebSocket)
- 캐싱 레이어 (Redis)
