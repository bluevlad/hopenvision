## 1. 배경

ADR-001 에서 academy `acm_basic` 통합을 결정. PHASE1_FLYWAY_PLAN.md 는 통합 범위로 **209 테이블 + 2 뷰 + 35 프로시저 + 18 함수** 의 1:1 포팅을 계획했다.

운영 가능한 통합 시스템을 즉시 만드는 것이 통합의 목표가 아님이 확인되었다. 진짜 목표는:

1. **평가/분석 AI 모델** 을 통한 신규 **합격예측 시스템** 완성
2. **EduFit** (학원/강사 평판·트렌드 분석) 의 학습 결과를 HopenVision 에 자동 연동 → 전문 AI 모델로 발전
3. Java 단일 → Java + React 이원화 모더나이제이션

이 관점에서 레거시 1:1 포팅의 ROI 를 재평가한다.

## 2. 결정

### 2.1 레거시 자산의 역할 재정의

| 자산 | 역할 | 정책 |
|---|---|---|
| **레거시 스키마 (DDL)** | AI 학습 모델의 입력 스키마 정의 | **이관 보존** (V2~V28) |
| **레거시 데이터 (행)** | AI 모델 학습/검증용 corpus | **ETL 보존** (Phase 3, 읽기 전용) |
| **레거시 비즈니스 로직 (프로시저 35개)** | 운영 재사용 가치 낮음 | **포팅 보류** — 필요 발생 시 신규 설계 |
| **레거시 함수 (18개)** | 일부는 단순 SQL 함수로 즉시 대체, 일부는 보안 위험 | **선택적 작성** (단순 9개만 필요 시) |
| **레거시 뷰 (2개)** | 통합 회원 뷰는 AI 입력 스키마로 유용 | **유지하되 V50 작성 보류** |

PR #481 머지 시점(2026-04-28) 의 실제 적용 상태를 반영하여 다음과 같이 정한다.

|---|---|---|---|---|

| V51 | 시퀀스 함수 7개 | ✅ **실구현 머지됨** | **유지** (ETL 시 호출처 검증 필요) | |

#### V53/V55 STUB 처리 정책

PR #481 머지된 V53, V55 STUB 파일은 **빈 껍데기 (CREATE FUNCTION ... RETURNS ... AS $$ BEGIN RETURN NULL; END; $$)** 형태로 prod 배포됨.

- 본 ADR 의 결정에 따라 **STUB 본문은 채우지 않는다** (영구 PASS-THROUGH).
- 호출 경로가 신규 모듈에서 발견되면 → STUB 호출이 아닌 **신규 Service / AI 모델 호출** 로 우회 설계.
- 향후 운영 화면 트리아지에서 STUB 호출 의존이 끊기면 **DROP** 후 별도 V 로 정리.

### 2.3 신규 트랙 우선순위

```
P0  레거시 데이터 ETL (Phase 3)             — AI 학습 corpus 확보
P0  AI 데이터 파이프라인 스켈레톤            — feature store, 모델 입출력 스키마
P1  EduFit ↔ HopenVision 연동 인터페이스    — 평판/트렌드 시그널 자동 인입
P1  합격예측 모델 v0 (baseline)             — 단순 회귀/규칙 기반부터 시작
P2  모델 자동 학습/배포 파이프라인          — 트렌드 변화에 따른 자동 업데이트
P3  레거시 함수/프로시저 선택적 작성        — 운영 화면에서 호출이 실제로 필요해질 때
```

## 3. 결과 (Consequences)

### 3.1 Positive

- 통합 마이그레이션 작업량 약 **40~50% 축소** (V52~V60 대부분 취소/보류)
- 신규 AI 모델 트랙에 리소스 집중 가능
- EduFit 통합 가치를 빨리 검증할 수 있음 (트렌드 시그널 → 모델 입력)

### 3.2 Negative / 트레이드오프

- 레거시 운영 화면 1:1 재현 불가 (애초 목적 아님)
- 함수/프로시저 호출이 깨진 채 남아있는 화면이 발견될 수 있음 → **Phase 4 운영 화면 트리아지에서 신규 설계로 전환** 또는 화면 자체를 폐기

### 3.3 Neutral

- V2~V28 (DDL) 은 그대로 진행. 레거시 스키마는 학습 자료 보존 목적상 유지됨.
- ETL (Phase 3) 은 그대로 필요.
